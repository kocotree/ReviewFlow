# ReviewFlow 评分域

ReviewFlow 接收提报人的需求材料，并在材料完整可读的前提下生成一次 AI 评分。

## Language

**需求文档**:
由多维表 `需求文档` 字段引用、可通过飞书官方 API 导出为 PDF 的一个或多个在线文档；当前包括 `doc/docx` 及最终指向它们的 Wiki 节点。文档按解析后的真实 token 去重并保留首次出现位置，其正文、截图、图表和排版共同构成评分内容。
_Avoid_: 纯文本文档、文档链接内容

**需求文档评分表示**:
一次评分中代表需求文档的唯一内容来源是本次实时导出的 PDF。系统不调用 `raw_content` 接口，也不使用在线文档纯文本或历史字段降级评分。
_Avoid_: 文档纯文本、PDF 加纯文本双份输入、历史字段降级

**不支持的在线文档**:
无法通过当前飞书 Drive v1 导出任务 API 导出为 PDF 的在线对象，包括 `sheet`、`bitable`、`slides`、`mindnote` 和普通文件；它们不能作为需求文档进入评分。
_Avoid_: 可降级文档、纯文本替代文档

**完整采集**:
一次评分所需的全部需求材料均已按当前评分模型要求成功取得。存在需求文档时必须成功取得其 PDF；`.doc/.docx/.txt/.md/.png/.jpg/.jpeg/.webp` 附件必须取得或成功转换为 PDF，否则不属于完整采集。
_Avoid_: 部分成功、可降级采集

**评分文件**:
全部在线文档及支持附件在转换完成后合并形成的单个总 PDF。在线需求文档按字段出现顺序固定在前，附件按多维表字段返回顺序排列在后；除原始描述仍作为文本外，评分只使用这一个总 PDF。
_Avoid_: 原格式附件、多份评分文件、图片模态、纯文本附件

**不支持附件**:
不在允许格式集合中的附件。一次提交中只要存在不支持附件，整次提交都不可评分，必须汇总提示后由用户删除或替换。
_Avoid_: 可跳过附件、部分评分附件

**附件**:
多维表 `需求附件` 字段中的文件型需求材料。附件按 `file_token` 去重并保留首次出现位置，重复项不进入下载、转换、合并或评分。
_Avoid_: 重复附件、附件副本

**无效附件**:
格式受支持但已损坏、加密或无法解析的附件。它属于用户可修复的材料问题，不能被跳过或降级处理。
_Avoid_: 技术异常附件、可忽略附件

**不可用需求文档**:
因无访问权限、链接失效、类型不支持或已删除而无法取得 PDF 的在线需求文档。它属于用户可修复的材料问题，不属于系统评分异常。
_Avoid_: 系统故障文档、管理员异常

**来源分隔页**:
总 PDF 中位于每份材料正文之前的标识页，记录材料类型以及在线文档标题或附件原文件名，使评分模型能够识别材料边界。
_Avoid_: 空白分页、技术元数据页

**文件能力模型**:
能够直接接收总 PDF，并据此评审正文、截图、图表和排版的评分模型。所有可评分提交都会形成总 PDF，因此纯文本模型不具备完成评分所需的能力。
_Avoid_: 文本模型、纯文本降级模型

**综合评分**:
将一次提报中的原始描述、需求文档和全部附件作为同一份需求材料，通过一次模型调用产生一个总分；不为各来源分别打分，也不计算来源平均分。
_Avoid_: 分项评分、附件平均分、文档平均分

**可评分提交**:
至少包含一份在线需求文档或一个支持的附件，两类材料不要求同时存在；同一次提交可以包含多个在线需求文档。原始描述只是补充文本，单独存在时不能触发评分。
_Avoid_: 纯描述提交、空文件提交

**原始描述**:
总 PDF 之外的补充评分文本，不属于文件型需求材料；它不进入总 PDF，且不能单独构成可评分提交。
_Avoid_: 主文档、PDF 封面

**完成评分**:
需求材料已完整采集，模型已生成有效综合评分，且 `AI评分`、`评分状态` 和隐藏的 `修改轮次` 已通过一次最终写回共同提交。评分详情、亮点和改进建议只用于本次通知。任一条件缺失都不算完成评分。
_Avoid_: 模型已返回、部分写回、详情落表

**评分异常**:
系统硬失败，或瞬时技术故障自动重试三次后仍失败的管理员处理状态。它不增加修改轮次，不向提交人发送卡片，仅允许管理员从群告警卡片重试。
_Avoid_: 用户材料问题、未通过、用户可重评状态

**评分写回**:
评分过程只写 `评分状态`；完成时原子写回 `AI评分`、`评分状态` 和隐藏的 `修改轮次`。`AI评分详情`、`AI评分时间` 与 `文档内容缓存` 不再属于系统字段，服务不读写也不负责清理历史值。
_Avoid_: 详情落表、时间落表、缓存落表、自动清理旧字段
